# Beyond Click — Consultation Summary (2026-04-16)

**Purpose:** Share the feedback from the consultation and clarify the next steps for the next iteration of the project.

**Project snapshot (what I presented):**
- Early digital experiments exploring new reading/browsing interactions: Tilt Scroll, Browse Trail Logger, Reading State Machine (plus a gaze-based page turning test).
- Three main concept directions combining physical + digital:
  - **Letter Box:** make the “no ads” intention actionable by filtering advertising mail.
  - **Mindful Candle:** use a candle/incense ritual as a gentle interface for intention, transitions, and reflection.
  - **Book Rest:** detect returning to a book and support re-entering focused reading; potentially extend reading into a spatial/augmented experience.

## Cross-cutting feedback (key themes)
- Start from an **applied context + purpose** (who/where/why) before technology; the same body signal can mean very different things depending on context.
- Distinguish **intent prediction / implicit interaction** from “body detection”, “explicit interaction”, or “automation/state machine”.
- Don’t force tangibility: place the **tangible interface only where it is justified** (input and/or output), not only because it is interesting.
- Narrow down using a **funnel method**: choose one context + one purpose and let every design decision align with it.

## Concept-by-concept feedback and response plan

### Tilt Scroll (digital experiment)
**Feedback**
- The prototype works and is clearly linked to the device itself (orientation/position).
- Currently it still reads as a relatively **explicit interaction** (especially for carousel navigation), and it is unclear how it becomes implicit/intent-based.

**Next steps**
- If carried forward: define a concrete applied context (e.g., commuting, one-handed reading, accessibility) and what the system should infer beyond “the device moved”.
- Explore a more implicit mapping (continuous assistance rather than discrete “commands”).

### Browse Trail Logger (digital experiment)
**Feedback**
- Logging/visualizing trails is a solid start, but it is still mostly “a report/database”.
- Missing “one more brick”: what new experience does this enable (beyond showing history)?

**Next steps**
- Define the purpose (e.g., help users recover a lost research thread, re-enter a rabbit hole intentionally, reflect on interests).
- Prototype 1–2 interventions (jump-back, resurfacing, clustering/summarizing) rather than only recording.

### Reading State Machine / Gaze Page Turner (digital experiment)
**Feedback**
- Risk of becoming a **state machine/automation** rather than intent prediction.
- Needs a clearer rationale: what is inferred, what changes for the user, and why it matters.

**Next steps**
- Keep only if it supports the chosen project purpose; otherwise treat as an early exploration and park it.
- If kept: specify behavior-derived states that reduce burden (without manual tagging) and validate with real reading workflows.

### Letter Box (main concept)
**Feedback**
- Context is funny/relatable, but the **human is out of the loop**: the system runs inside the mailbox without meaningful user action.
- Not clearly linked to the “mind-reading / intent” brief: filtering ads vs letters is not the same as inferring what the user values.
- Consider how preferences are expressed through user actions (sorting piles, what stays untouched for weeks, keep/discard actions).

**Next steps**
- Reframe to include user signals and preference learning so the system responds to *my* intention (not just paper classification).
- Map where tangibility is justified (input, output, or both) and what role the user plays.

### Mindful Candle (main concept)
**Feedback**
- Strong base: creating a new habit often works best when building on an existing ritual.
- Current trigger is too binary (“on/off”), close to a switch; lacks nuance.
- Technology risks feeling like a big add-on that breaks the calm context (a screen/object appearing “out of nowhere”).
- Needs a clearer scenario: what comes after lighting the candle? What is the system predicting/assisting (beyond automation)?

**Next steps**
- List micro-actions and variations inside the self-care/ritual context (finer signals than “candle on”).
- Explore alternatives where the candle is **output** (ambient cue) rather than only input.
- Prototype subtle outputs (light/sound/ambient feedback) that preserve comfort; avoid screen-first solutions unless justified.

### Book Rest (main concept)
**Feedback**
- Seen as more aligned with implicit interaction and closer to the brief.
- Tangible still needs justification; sensing can happen elsewhere (e.g., webcam), so the object’s role must be intentional.

**Next steps**
- Focus on the “returning to reading” threshold: detect picking up the book and support re-entry into focused reading.
- Prototype 1 minimal assistance behavior (ambient light or remembered context cue) before moving to projection/spatial augmentation.

## Proposed next steps (next 1–2 weeks)
- Write a one-sentence **project purpose + applied context** (who/where/why) and use it as a filter for all concepts.
- Decide on **1 primary direction** (and 1 backup) to develop deeper; reduce parallel prototyping.
- For the chosen direction: storyboard a concrete scenario, define “signals → inference → response”, and specify where the human is in the loop.
- Build the next prototype iteration focused on implicit interaction + subtle, context-fitting output.
